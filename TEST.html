Structure

To include the IR in my DBI engine, I have to choice a particular layout / design. I distinguish two designs:

  • A lifter on function granularity, it requires to lift the whole function. But the optimizations pass are easier to achieve because the code is entirely lifted.
  • A lifter on basic block granularity, that's harder to develop but it's the most reliable option since we just have to instrument the guest program basic block per basic block. That's harder because if we want to apply some optimizations pass, we have to reconstruct the function according to the basic blocks we've reached, and from those we don't reach. That's also the slower option.
